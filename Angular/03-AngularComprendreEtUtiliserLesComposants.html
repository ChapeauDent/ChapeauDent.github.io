Comprendre et Utiliser les Composants dans Angular

Comprendre la notion de composant

Les composants sont l'un des concepts centraux d'Angular. L'interface de l'application est constituée d'une arborescence de composants. Un composant est un bloc de code réutilisable, ce qui permet d'emboîter ces composants les uns dans les autres. Par exemple, une application Angular commence par un composant racine, appelé AppComponent, qui représente l'application entière. Ce composant contiendra d'autres sous-composants comme le menu, le corps de la page ou une barre de navigation (NavBar). Ces sous-composants peuvent eux-mêmes contenir d'autres sous-composants, formant ainsi une structure hiérarchique.

En résumé, un composant Angular représente un morceau d'interface de l'application. Un composant peut aller d'un simple bouton à une page web entière. Cette modularité offre une grande flexibilité pour construire des interfaces riches et complexes.

Chaque composant est autonome et possède ses propres données et méthodes, ce qui facilite le développement modulaire. Les composants peuvent également être partagés entre différentes parties de l'application, ce qui améliore la réutilisabilité du code et la maintenabilité.

La structure d'un composant Angular

Pour mieux comprendre comment fonctionnent les composants, il est important de se familiariser avec leur structure. Chaque composant Angular est constitué de plusieurs fichiers :

  • HTML : Fichier contenant le code de l'interface utilisateur du composant. Par exemple, app.component.html.
  • CSS : Fichier contenant les styles du composant, par exemple app.component.css. Chaque composant Angular possède son propre fichier de style, ce qui permet d'appliquer des styles à l'échelle locale du composant.
  • TypeScript (TS) : Fichier qui contient la logique du composant, par exemple app.component.ts. Ce fichier est écrit en TypeScript et contient le code permettant de gérer les comportements de l'interface.
  • Fichier de test (spec.ts) : Utilisé pour les tests unitaires du composant. Bien que nous n'utiliserons pas ces fichiers dans ce cours, ils sont importants dans le cadre de développement d'applications robustes.

Un composant Angular est généralement construit à l'aide d'un décorateur @Component, qui relie les différents fichiers ensemble. Le décorateur contient notamment :

  • selector : Le nom de la balise HTML pour afficher ce composant, par exemple app-root.
  • templateUrl : Le chemin vers le fichier HTML du composant.
  • styleUrls : Le chemin vers le fichier CSS du composant.

Voici un exemple de décorateur de composant :

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  title = 'Mon Application Angular';
}

Les composants peuvent également être enrichis avec des Input et Output pour la communication entre les composants. Les propriétés @Input() permettent de recevoir des données d'un composant parent, tandis que @Output() permet d'envoyer des événements à un composant parent.

Le cycle de vie d'un composant

Chaque composant Angular passe par plusieurs états au cours de son existence, appelés cycle de vie du composant. Cela inclut la création, la mise à jour et la destruction du composant. Angular propose des hooks de cycle de vie permettant aux développeurs d'exécuter du code à différents moments de la vie d'un composant.

Voici quelques hooks de cycle de vie importants :

  • ngOnChanges : Appelé lors de la modification des données d'entrée d'un composant.
  • ngOnInit : Appelé une fois lors de l'initialisation du composant. Idéal pour effectuer des tâches comme la récupération de données à partir d'un service.
  • ngOnDestroy : Appelé juste avant la destruction du composant, utile pour nettoyer des ressources (comme des abonnements).

Voici un exemple d'utilisation du hook ngOnInit :

import { Component, OnInit } from '@angular/core';

@Component({
  selector: 'app-article',
  templateUrl: './article.component.html',
  styleUrls: ['./article.component.css']
})
export class ArticleComponent implements OnInit {
  constructor() { }

  ngOnInit(): void {
    console.log('Le composant Article est initialisé');
  }
}

Comprendre le cycle de vie des composants est essentiel pour optimiser la gestion des ressources et s'assurer que les actions appropriées sont effectuées au bon moment (par exemple, initialiser des données, se désabonner de services).

Créer un Composant Angular

Pour apprendre à créer un composant Angular, nous allons détailler chaque étape en partant de l'exemple fourni dans la documentation officielle Angular. Voici comment procéder :

Étape 1 : Créer un Nouveau Composant

Pour créer un composant Angular, tu utilises la CLI d’Angular qui te permet de générer automatiquement les fichiers nécessaires. Ouvre ton terminal et exécute la commande suivante :

ng generate component user-profile

Cette commande génère un composant appelé UserProfileComponent avec les fichiers suivants :

  • user-profile.component.html : Fichier contenant le template HTML du composant.
  • user-profile.component.css : Fichier contenant le style du composant.
  • user-profile.component.ts : Fichier contenant la logique du composant en TypeScript.
  • user-profile.component.spec.ts : Fichier utilisé pour les tests unitaires.

Étape 2 : Comprendre la Structure du Composant

Voici un exemple simple du fichier user-profile.component.ts pour comprendre la structure de base d'un composant Angular :

import { Component } from '@angular/core';

@Component({
  selector: 'app-user-profile',
  templateUrl: './user-profile.component.html',
  styleUrls: ['./user-profile.component.css']
})
export class UserProfileComponent {
  name: string = 'John Doe';
  age: number = 30;
  bio: string = 'Développeur passionné par Angular et les technologies web.';
}

Voici les parties importantes à comprendre :

  • Importation du décorateur `@Component` : Importation du décorateur `@Component` depuis le module `@angular/core`. Ce décorateur est essentiel pour déclarer à Angular que cette classe est un composant.
  • Décorateur `@Component` : Le décorateur contient des propriétés telles que :
    • `selector` : Le nom de la balise HTML utilisée pour insérer ce composant dans d'autres templates. Ici, la balise sera ``.
    • `templateUrl` : Spécifie le fichier HTML contenant le template de ce composant.
    • `styleUrls` : Spécifie le fichier CSS contenant les styles associés à ce composant.
  • Classe `UserProfileComponent` : La classe contient des propriétés qui représentent les données du composant. Par exemple, `name`, `age`, et `bio`.

Étape 3 : Créer le Template HTML

Dans le fichier user-profile.component.html, crée la vue qui sera rendue par ton composant :

<div class="user-profile">
  <h2>{{ name }}</h2>
  <p>Âge : {{ age }}</p>
  <p>Bio : {{ bio }}</p>
</div>

Ici, tu utilises la syntaxe `{{ }}` pour lier les propriétés définies dans le composant TypeScript (comme `name`, `age`, `bio`) à la vue. Cela s'appelle la liaison de données ou data binding.

Étape 4 : Ajouter des Styles

Dans le fichier user-profile.component.css, ajoute des styles spécifiques à ce composant :

.user-profile {
  border: 1px solid #ccc;
  padding: 16px;
  border-radius: 8px;
  max-width: 300px;
  margin: 16px auto;
  text-align: center;
}

.user-profile h2 {
  font-size: 1.5em;
  color: #333;
}

Ces styles sont encapsulés et n'affecteront que ce composant.

Étape 5 : Intégrer le Composant dans l'Application

Pour intégrer ce nouveau composant dans une autre partie de ton application, par exemple dans app.component.html (le composant principal), ajoute la balise définie par le `selector` de ton composant :

<app-user-profile></app-user-profile>

Cela inclut le composant UserProfileComponent dans le composant racine de ton application.

Étape 6 : Exemple Complet

user-profile.component.ts

import { Component } from '@angular/core';

@Component({
  selector: 'app-user-profile',

})
export class UserProfileComponent {
  name: string = 'John Doe';
  age: number = 30;
  bio: string = 'Développeur passionné par Angular et les technologies web.';
}

user-profile.component.html

<div class="user-profile">
  <h2>{{ name }}</h2>
  <p>Âge : {{ age }}</p>
  <p>Bio : {{ bio }}</p>
</div>

Étape 7 : Ajouter une Interaction avec des Propriétés `@Input`

Pour rendre ce composant plus dynamique, utilise `@Input` pour permettre au composant de recevoir des données depuis un composant parent :

user-profile.component.ts (avec `@Input`)

import { Component, Input } from '@angular/core';

@Component({
  selector: 'app-user-profile',
  templateUrl: './user-profile.component.html',
  styleUrls: ['./user-profile.component.css']
})
export class UserProfileComponent {
  @Input() name: string = 'John Doe';
  @Input() age: number = 30;
  @Input() bio: string = 'Développeur passionné par Angular et les technologies web.';
}

Maintenant, lorsque tu veux intégrer ce composant dans app.component.html, tu peux passer des valeurs différentes à chaque instance :

<app-user-profile name="Alice" age="25" bio="Designer créatif passionné par l'UI/UX."></app-user-profile>
<app-user-profile name="Bob" age="40" bio="Chef de projet expert en Agile et en gestion d'équipe."></app-user-profile>

Avec cette approche, tu peux réutiliser le même composant tout en ayant des données différentes, ce qui le rend très puissant pour créer des interfaces dynamiques et modulaires.
