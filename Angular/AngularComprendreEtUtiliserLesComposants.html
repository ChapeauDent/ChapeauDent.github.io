Utiliser les Composants dans Angular

Comprendre la notion de composant

Les composants sont l'un des concepts centraux d'Angular. L'interface de l'application est constituée d'une arborescence de composants. Un composant est un bloc de code réutilisable, ce qui permet d'emboîter ces composants les uns dans les autres. Par exemple, une application Angular commence par un composant racine, appelé AppComponent, qui représente l'application entière. Ce composant contiendra d'autres sous-composants comme le menu, le corps de la page ou une barre de navigation (NavBar). Ces sous-composants peuvent eux-mêmes contenir d'autres sous-composants, formant ainsi une structure hiérarchique.

En résumé, un composant Angular représente un morceau d'interface de l'application. Un composant peut aller d'un simple bouton à une page web entière. Cette modularité offre une grande flexibilité pour construire des interfaces riches et complexes.

La structure d'un composant Angular

Pour mieux comprendre comment fonctionnent les composants, il est important de se familiariser avec leur structure. Chaque composant Angular est constitué de plusieurs fichiers :

  • HTML : Fichier contenant le code de l'interface utilisateur du composant. Par exemple, app.component.html.
  • CSS : Fichier contenant les styles du composant, par exemple app.component.css. Chaque composant Angular possède son propre fichier de style, ce qui permet d'appliquer des styles à l'échelle locale du composant.
  • TypeScript (TS) : Fichier qui contient la logique du composant, par exemple app.component.ts. Ce fichier est écrit en TypeScript et contient le code permettant de gérer les comportements de l'interface.
  • Fichier de test (spec.ts) : Utilisé pour les tests unitaires du composant. Bien que nous n'utiliserons pas ces fichiers dans ce cours, ils sont importants dans le cadre de développement d'applications robustes.

Un composant Angular est généralement construit à l'aide d'un décorateur @Component, qui relie les différents fichiers ensemble. Le décorateur contient notamment :

  • selector : Le nom de la balise HTML pour afficher ce composant, par exemple app-root.
  • templateUrl : Le chemin vers le fichier HTML du composant.
  • styleUrls : Le chemin vers le fichier CSS du composant.

Voici un exemple de décorateur de composant :

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  title = 'Mon Application Angular';
}

Le cycle de vie d'un composant

Chaque composant Angular passe par plusieurs états au cours de son existence, appelés cycle de vie du composant. Cela inclut la création, la mise à jour et la destruction du composant. Angular propose des hooks de cycle de vie permettant aux développeurs d'exécuter du code à différents moments de la vie d'un composant.

Voici quelques hooks de cycle de vie importants :

  • ngOnChanges : Appelé lors de la modification des données d'entrée d'un composant.
  • ngOnInit : Appelé une fois lors de l'initialisation du composant. Idéal pour effectuer des tâches comme la récupération de données à partir d'un service.
  • ngOnDestroy : Appelé juste avant la destruction du composant, utile pour nettoyer des ressources (comme des abonnements).

Voici un exemple d'utilisation du hook ngOnInit :

import { Component, OnInit } from '@angular/core';

@Component({
  selector: 'app-article',
  templateUrl: './article.component.html',
  styleUrls: ['./article.component.css']
})
export class ArticleComponent implements OnInit {
  constructor() { }

  ngOnInit(): void {
    console.log('Le composant Article est initialisé');
  }
}

Créer un nouveau composant

Pour créer un nouveau composant dans Angular, vous pouvez utiliser la commande Angular CLI suivante :

ng generate component nom-du-composant

Par exemple, pour créer un composant article :

ng generate component article

Cette commande créera un dossier article contenant les fichiers nécessaires : article.component.html, article.component.css, article.component.ts, et article.component.spec.ts. Vous pourrez alors personnaliser chaque fichier pour construire votre interface.

Intégrer un composant dans l'application

Une fois un composant créé, vous pouvez l'intégrer à d'autres composants en utilisant sa balise HTML définie par le selector. Par exemple, si le selector de votre composant article est app-article, vous pouvez l'ajouter au composant principal (app.component.html) comme suit :

<app-article></app-article>

Vous pouvez répéter cette balise autant de fois que nécessaire pour afficher plusieurs instances du composant.

Personnaliser un composant

Un composant est un bloc de code réutilisable et personnalisable. Pour personnaliser l'apparence d'un composant, vous pouvez modifier son fichier CSS. Chaque composant possède son propre fichier de style, garantissant que les styles ne se propagent pas en dehors du composant.

Pour appliquer des styles à l'ensemble de l'application, vous pouvez utiliser le fichier styles.css situé à la racine du projet. Cela permet d'appliquer des règles de style à tous les composants.

Comprendre les modules dans Angular

Angular utilise des modules pour structurer l'application. Chaque application Angular a au moins un module principal, app.module.ts. Un module regroupe des composants, des services, des directives, etc., et définit leur portée.

Le décorateur @NgModule est utilisé pour définir un module. Il inclut plusieurs propriétés, telles que :

  • declarations : Liste des composants appartenant à ce module.
  • imports : Liste des modules dont ce module dépend.
  • providers : Définit les services disponibles dans ce module.
  • bootstrap : Définit le composant racine à lancer au démarrage de l'application.
