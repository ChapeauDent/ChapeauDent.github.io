Découverte d'Angular

Qu'est-ce que le framework Angular ?

Angular est un framework open source frontend de JavaScript développé par Google. Historiquement, Angular a évolué à partir d'AngularJS lancé en 2010, qui souffrait d'une syntaxe complexe et de limitations liées à JavaScript. En 2016, Google a lancé une version totalement repensée appelée Angular ou Angular 2, à ne pas confondre avec AngularJS. Angular est aujourd'hui l'un des frameworks les plus populaires pour le développement web.

Pourquoi utiliser Angular ?

Le principal atout d'Angular est son architecture de type MVC (Modèle - Vue - Contrôleur), qui divise la structure du projet en trois parties :

  • Modèle : Responsable de la gestion des données, comme la récupération d'informations depuis une base de données.
  • Vue : Gère l'interface graphique, comme le template HTML.
  • Contrôleur : Gère la logique de l'application et fait le lien entre le Modèle et la Vue.

Angular utilise TypeScript, un sur-ensemble de JavaScript qui apporte des fonctionnalités de typage stricte, offrant un développement plus stable, rapide et sécurisé. Angular permet aussi de créer des Single Page Applications (SPA), où une seule page HTML est modifiée dynamiquement sans rechargement complet, améliorant l'expérience utilisateur.

Liaison de données bidirectionnelle

Angular se distingue par sa liaison de données bidirectionnelle, c'est-à-dire une synchronisation instantanée entre le Modèle et la Vue. Cela permet de mettre à jour automatiquement l'interface lorsque les données changent, et inversement.

Différence entre une library et un framework

Angular est un framework, mais qu'est-ce qu'un framework, et en quoi diffère-t-il d'une library (bibliothèque) ?

Library

Une library est un ensemble de fonctionnalités que l'on peut utiliser selon nos besoins. Elle ne définit pas l'architecture du projet, laissant la liberté au développeur de choisir ce qu'il souhaite utiliser.

Framework

Un framework, quant à lui, impose une architecture globale. Le développeur doit insérer son code dans cette structure, et c'est le framework qui appelle le code du développeur en fonction de ses propres règles. Angular, en tant que framework, offre une base solide et pré-structurée pour créer des applications robustes.

Découvrir TypeScript

Angular est basé sur TypeScript, un sur-ensemble de JavaScript. TypeScript est transcompilé en JavaScript, ce qui signifie qu'il est converti en JavaScript exécutable après compilation.

TypeScript apporte des fonctionnalités telles que le typage strict, qui permet de s'assurer que les variables et les valeurs respectent un type précis, rendant le code plus sûr et plus facile à maintenir. Il offre aussi une syntaxe orientée objet et permet l'utilisation de classes et interfaces, facilitant le développement modulaire et la réutilisation du code.

Installer l'environnement de développement

Pour travailler avec Angular, il est nécessaire de configurer un environnement de développement complet. Voici les étapes et les éléments à installer, avec des explications sur l'utilité de chaque composant :

  1. IDE (Environnement de développement intégré) - Visual Studio Code

    2. Un IDE est un outil essentiel pour écrire, organiser et tester du code. Visual Studio Code (VS Code) est un des IDE les plus populaires pour le développement Angular, car il est léger, gratuit, et possède de nombreuses extensions qui facilitent le travail avec Angular et TypeScript. Pour l'installer, rendez-vous sur code.visualstudio.com et suivez les instructions.

  2. Node.js

    3. Node.js est un environnement d'exécution JavaScript qui permet de lancer des outils en ligne de commande nécessaires au développement Angular. Par exemple, Angular CLI utilise Node.js pour exécuter des commandes qui gèrent les dépendances, génèrent des composants, ou démarrent des serveurs de développement. Pour installer Node.js, allez sur nodejs.org et téléchargez la version recommandée.

  3. npm (Node Package Manager)

    4. Avec l'installation de Node.js, vous obtiendrez également npm. npm est un gestionnaire de paquets qui permet d'installer des librairies et des outils nécessaires au développement de votre application. Vous utiliserez npm pour installer Angular CLI ainsi que d'autres dépendances utiles.

  4. Angular CLI (Command Line Interface)

    5. Angular CLI est un outil de ligne de commande qui simplifie la gestion des projets Angular. Il permet de créer, de configurer et de développer des applications Angular directement depuis le terminal.rendez-vous sur Angular.io pour obtenir plus d'information. Pour l'installer, ouvrez un terminal et exécutez la commande suivante :

    npm install -g @angular/cli

    Le paramètre -g signifie que l'installation est globale, c'est-à-dire qu'Angular CLI sera disponible pour tous vos projets.

Une fois l'environnement configuré, vous pourrez utiliser Angular CLI pour créer un nouveau projet en tapant :

ng new nom-du-projet

Cette commande crée la structure de base de votre projet Angular et installe automatiquement toutes les dépendances nécessaires. Vous serez alors prêt à commencer le développement.
