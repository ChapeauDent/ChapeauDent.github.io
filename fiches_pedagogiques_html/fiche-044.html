Compétence : Choisir le type d’entrée pertinent (type="email", type="date", etc.)

Date : 20/06/2025

Version : v1.0

Objectif

Comprendre comment choisir le type d’entrée pertinent dans les formulaires HTML.

Théorie

Les formulaires HTML permettent de collecter des données utilisateur. Pour améliorer l'expérience utilisateur et la validation des données, il est important de choisir le type d'entrée approprié pour chaque champ de formulaire. Voici quelques types d'entrées courants :

  • type="text" : Champ de texte simple.
  • type="email" : Champ pour saisir une adresse e-mail. Valide automatiquement le format de l'e-mail.
  • type="date" : Champ pour sélectionner une date. Affiche un sélecteur de date.
  • type="number" : Champ pour saisir un nombre. Permet de définir des valeurs minimales et maximales.
  • type="password" : Champ pour saisir un mot de passe. Masque les caractères saisis.
  • type="url" : Champ pour saisir une URL. Valide automatiquement le format de l'URL.
  • type="tel" : Champ pour saisir un numéro de téléphone. Peut appliquer un masque de saisie.

Exemple

Voici un exemple de formulaire HTML utilisant différents types d'entrées :

<form>
    <label for="email">Email :</label>
    <input type="email" id="email" name="email" required>
    <br>
    <label for="date">Date de naissance :</label>
    <input type="date" id="date" name="date" required>
    <br>
    <label for="number">Nombre d'enfants :</label>
    <input type="number" id="number" name="number" min="0" max="10">
    <br>
    <label for="password">Mot de passe :</label>
    <input type="password" id="password" name="password" required>
    <br>
    <label for="url">Site web :</label>
    <input type="url" id="url" name="url">
    <br>
    <label for="tel">Téléphone :</label>
    <input type="tel" id="tel" name="tel">
    <br>
    <button type="submit">Envoyer</button>
</form>
