Compétence : Gérer les erreurs de saisie avec les attributs HTML (required, pattern, etc.)

Date : 20/06/2025

Version : v1.0

Objectif

Comprendre comment gérer les erreurs de saisie dans les formulaires HTML à l'aide des attributs comme required, pattern, etc.

Théorie

Les formulaires HTML permettent de collecter des données auprès des utilisateurs. Pour garantir la qualité et la validité des données saisies, il est essentiel de gérer les erreurs de saisie. HTML offre plusieurs attributs pour ce faire :

  • required : Cet attribut indique que le champ doit être rempli avant la soumission du formulaire.
  • pattern : Cet attribut permet de définir une expression régulière que la valeur du champ doit respecter.
  • min et max : Ces attributs définissent les valeurs minimales et maximales pour les champs de type numérique.
  • maxlength : Cet attribut définit le nombre maximal de caractères que le champ peut contenir.
  • type : Cet attribut permet de spécifier le type de données attendu (email, url, number, etc.).

Exemple

Voici un exemple de formulaire HTML utilisant les attributs de validation :

<!DOCTYPE html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Exemple de Formulaire</title>
</head>
<body>
    <form>
        <label for="nom">Nom :</label>
        <input type="text" id="nom" name="nom" required maxlength="50">
        <br>
        <label for="email">Email :</label>
        <input type="email" id="email" name="email" required>
        <br>
        <label for="age">Âge :</label>
        <input type="number" id="age" name="age" min="18" max="99" required>
        <br>
        <label for="site">Site Web :</label>
        <input type="url" id="site" name="site">
        <br>
        <label for="motdepasse">Mot de passe :</label>
        <input type="password" id="motdepasse" name="motdepasse" required pattern=".{8,}">
        <br>
        <button type="submit">Envoyer</button>
    </form>
</body>
</html>
