Compétence : Choisir le bon formulaire selon le contexte

Date : 20/06/2025

Version : v1.0

Objectif

Comprendre comment choisir le bon formulaire HTML selon le contexte d'utilisation.

Théorie

Les formulaires HTML sont utilisés pour collecter des données auprès des utilisateurs. Le choix du bon formulaire dépend du type de données à collecter et du contexte d'utilisation. Voici quelques exemples :

  • Formulaire de contact : Utilisé pour permettre aux utilisateurs de vous envoyer des messages. Il inclut généralement des champs pour le nom, l'adresse e-mail, et le message.
  • Formulaire d'inscription : Utilisé pour enregistrer les utilisateurs sur un site web. Il inclut des champs pour le nom, l'adresse e-mail, le mot de passe, et parfois des informations supplémentaires comme l'adresse.
  • Formulaire de recherche : Utilisé pour permettre aux utilisateurs de rechercher des informations sur un site web. Il inclut généralement un champ de texte pour la requête de recherche.
  • Formulaire de commande : Utilisé pour collecter des informations de commande auprès des utilisateurs. Il inclut des champs pour les informations de livraison, les détails de paiement, et les articles commandés.

Exemple

Voici un exemple de formulaire de contact :


<form action="/submit_contact" method="post">
    <label for="name">Nom</label>
    <input type="text" id="name" name="name" required>
    <br>
    <label for="email">Email</label>
    <input type="email" id="email" name="email" required>
    <br>
    <label for="message">Message</label>
    <textarea id="message" name="message" required></textarea>
    <br>
    <button type="submit">Envoyer</button>
</form>
