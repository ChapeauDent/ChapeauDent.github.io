Compétence : Préparer une page pour le responsive design

Date : 20/06/2025

Version : v1.0

Objectif

Comprendre comment préparer une page pour le responsive design en utilisant la balise viewport et une structure fluide.

Théorie

Le responsive design est une approche de conception web qui permet aux pages web de s'adapter à différentes tailles d'écran et résolutions. Cela garantit une expérience utilisateur optimale sur tous les appareils, qu'il s'agisse de smartphones, de tablettes ou d'ordinateurs de bureau.

Pour préparer une page pour le responsive design, il est essentiel d'utiliser la balise <meta name="viewport"> dans l'en-tête du document HTML. Cette balise permet de contrôler la mise en page et le zoom de la page sur les appareils mobiles.

Voici un exemple de balise viewport :

<meta name="viewport" content="width=device-width, initial-scale=1.0">

En plus de la balise viewport, il est important d'utiliser une structure fluide pour les éléments de la page. Cela signifie que les éléments doivent utiliser des unités relatives comme les pourcentages (%) plutôt que des unités fixes comme les pixels (px). Cela permet aux éléments de s'ajuster automatiquement à la taille de l'écran.

Exemple

Voici un exemple de code HTML qui utilise la balise viewport et une structure fluide :

<!DOCTYPE html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Exemple de Responsive Design</title>
    <style>
        body {
            font-family: Arial, sans-serif;
            margin: 0;
            padding: 0;
        }
        .container {
            width: 100%;
            padding: 20px;
            box-sizing: border-box;
        }
        .header, .content, .footer {
            background-color: #f4f4f4;
            margin-bottom: 20px;
            padding: 20px;
            border-radius: 8px;
        }
    </style>
</head>
<body>
    <div class="container">
        <div class="header">
            <h1>En-tête</h1>
        </div>
        <div class="content">
            <p>Contenu principal</p>
        </div>
        <div class="footer">
            <p>Pied de page</p>
        </div>
    </div>
</body>
</html>
